# MDTP 机器可读输出（与 Obsidian 插件「扩写」第 3 步对接，必须遵守）

在输出上述人类可读内容之后，**再追加**以下两段，用**完全相同**的标记行分割（行首无空格）：

1. 第一行：`<<<VIDEO_SCRIPT>>>`
2. 人读短视频脚本、选题、卡片拆分等（Markdown）
3. 第二行：`<<<VIDEO_CONFIG>>>`
4. 紧跟 **一个 JSON 对象**（可整体用 ` ```json ` 代码块包裹，也可裸写 JSON，但不得夹杂说明文字在 JSON 外）

**时长**：口播主稿以约 **30 秒** 为目标（全汉语约 90～120 字，按约 3.5～4 字/秒口播感估算）；平台结尾句可更短并利于收藏/关注。

**JSON 字段**（可扩展，但以下键名不得改名）：

- `accountInfo`：账号展示名
- `topic`：本视频主题/关键词
- `voiceover`：主配音**全文**（将用于 TTS 驱动中间图文主体时长；**不含**三平台差异化开头/结尾的完整重复，中间讲清方法为主）
- `platforms`：对象，必须包含三键 `douyin`、`xiaohongshu`、`shipinhao`，每键含：
  - `cover_title`：封面/片头/视频内标题用短句
  - `opening_text`：前 1～2 句抓人（可入首帧/口播）
  - `ending_text`：关注/收藏/转发的引导（可入尾帧/口播）
  - `publish_title`：对应平台**发布页**填写的**短标题**（可与 `cover_title` 相同或更短、更合规；抖音/视频号/小红书发作品时必用）
  - `publish_description`：对应平台**作品描述/简介/正文**（可含 1～3 个话题建议，**不是**全片逐字稿；供用户复制到发布页）
